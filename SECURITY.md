# Security Policy

## Supported Versions

The Fidan project aims to maintain security for actively developed versions of the language and its tooling.

Security fixes will generally target the latest stable development branch.

Older versions may not receive security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability in Fidan, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, report vulnerabilities privately so they can be investigated and fixed before public disclosure.

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the vulnerability
- A proof-of-concept if possible
- The affected Fidan version
- Your environment (OS, architecture, etc.)

This information helps maintainers quickly assess and resolve the issue.

---

## Responsible Disclosure

After a vulnerability is reported:

1. The maintainers will investigate the issue.
2. A fix will be developed if necessary.
3. The vulnerability may be publicly disclosed after a fix is available.

We appreciate responsible disclosure that allows issues to be addressed before they are widely known.

---

## Security Scope

Security issues may include but are not limited to:

- Memory safety vulnerabilities
- Compiler or runtime vulnerabilities
- Interpreter sandbox escapes
- Unsafe behavior that could lead to arbitrary code execution
- Vulnerabilities in official tooling or integrations

---

## Acknowledgements

We appreciate responsible security research and contributions that help improve the safety and reliability of the Fidan ecosystem.
